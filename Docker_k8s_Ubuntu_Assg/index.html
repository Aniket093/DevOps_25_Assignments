DevOps Project: HTML → Docker → Kubernetes

End-to-End Deployment of a Static Web Application

Project Overview

This project demonstrates how to build, containerize, and deploy a simple static web application using modern DevOps tools and practices.

Step 1: Create Static Web Application

  • Develop a basic HTML web page
  • Ensure content loads correctly in browser
  • No backend required (pure static content)

Step 2: Containerization using Docker

  • Create a Dockerfile using Nginx base image
  • Copy HTML files into Nginx web directory
  • Build Docker image
  • Run container locally

Step 3: Test Locally

  • Run container using Docker
  • Access application via localhost
  • Verify correct rendering

Step 4: Push to Docker Hub

  • Tag Docker image
  • Login to Docker Hub
  • Push image to remote repository

Step 5: Setup Kubernetes Cluster

  • Use Minikube or any Kubernetes setup
  • Start cluster and verify nodes

Step 6: Kubernetes Deployment

  • Create Deployment YAML
  • Define replicas and container image
  • Apply configuration using kubectl

Step 7: Expose Application

  • Create Service YAML (NodePort)
  • Expose application externally
  • Alternatively use port forwarding

Final Result

The application is successfully deployed on Kubernetes and accessible via browser, demonstrating a complete DevOps workflow from development to deployment.
